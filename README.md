# Тестовое задание: C++ Разработчик (Стажер)

Данный проект содержит готовое решение тестового задания, разделенное на модули:
1. **library**: Динамическая библиотека `libdynamic_lib.so` с тремя требуемыми функциями.
2. **program1**: Консольный клиент с двухпоточной архитектурой (`std::thread`, `std::mutex`, `std::condition_variable`).
3. **program2**: Консольный TCP-сервер, обрабатывающий полученные данные сокетов.

---

## 🚀 Запуск через Docker (Без необходимости устанавливать Linux локально)

Поскольку у вас установлен Docker, вы можете собрать и запустить проект в изолированной среде Linux за пару простых шагов.

### Вариант 1. Запуск через Docker Compose (Рекомендуемый)

Мы настроили специальный файл конфигурации, который запускает оба контейнера в общем сетевом пространстве. Это позволяет Программе №1 беспрепятственно общаться с Программой №2 по адресу `127.0.0.1`.

1. **Перейдите в директорию проекта в консоли:**
   ```bash
   cd cpp_project
   ```

2. **Соберите и запустите Программу №2 (Сервер):**
   ```bash
   docker compose up --build program2
   ```
   *Вы увидите лог ожидания подключений:* `Ожидание подключений от Программы №1 на порту 5001...`

3. **В новом окне терминала запустите интерактивную Программу №1 (Клиент):**
   ```bash
   docker compose run --rm program1
   ```
   *Теперь вы можете вводить числа непосредственно в консоль, наблюдать за синхронизацией потоков и работой библиотеки в реальном времени!*

---

### Вариант 2. Запуск вручную через стандартный Docker CLI

Если вы предпочитаете не использовать `compose`, соберите образ и запустите контейнеры вручную:

1. **Сборка единого Docker-образа:**
   ```bash
   docker build -t cpp-sandbox .
   ```

2. **Запуск Программы №2 (Сервер):**
   ```bash
   docker run --rm -p 5001:5001 --name p2 cpp-sandbox ./build/bin/program2
   ```

3. **Запуск Программы №1 (Клиент) в общем сетевом пространстве сервера:**
   ```bash
   docker run --rm -it --network="container:p2" cpp-sandbox ./build/bin/program1
   ```

## 🛠️ Описание функций библиотеки (из задания)

* **Функция 1 (`dynamic_lib`):** Сортирует строку из цифр по убыванию. Четные числа заменяются на английские буквы `KB`.
* **Функция 2 (`dynamic_lib`):** Возвращает сумму всех оставшихся численных элементов в строке (после замены четных чисел на `KB` остаются только нечетные цифры).
* **Функция 3 (`dynamic_lib`):** Анализирует сумму. Возвращает `true`, если сумма больше 2 и кратна 32.

## 📂 Структура репозитория

* `/library` — исходный код динамической библиотеки и `CMakeLists.txt`.
* `/program1` — исходный код клиента, реализация синхронизации потоков, мьютексов и `CMakeLists.txt`.
* `/program2` — исходный код сервера, обработка TCP-соединений и `CMakeLists.txt`.
* `CMakeLists.txt` — корневой конфигурационный файл для сборки всего проекта.
* `Dockerfile` — инструкция сборки Linux-окружения.
* `docker-compose.yml` — оркестрация запуска двух программ.
